# 生产工艺流程追溯系统设计规范

> 本文档只约束管理端的视觉语言和通用交互，不规定具体页面、菜单、业务模块或状态字典。
> 页面功能和业务流程以最新需求、API 契约和数据库设计为准。

## 1. 设计定位

### 1.1 风格

**Clean Industrial Admin / 清爽型工业管理后台**

关键词：清爽、克制、表格优先、弹窗表单、状态清晰、适中的信息密度、低装饰、低动效。

不采用大屏炫酷风、重型 ERP 灰蓝风、官网营销风，以及过度科技感或阴影。

## 2. 色彩规范

| 用途     | 色值      | 说明                       |
| -------- | --------- | -------------------------- |
| 主色     | `#283A50` | 侧边栏、标题强调、核心导航 |
| 辅助蓝   | `#306188` | 主按钮、链接、选中状态     |
| 强调黄   | `#EAB321` | 少量步骤或重点强调         |
| 页面背景 | `#F5F7FA` | 页面整体背景               |
| 卡片背景 | `#FFFFFF` | 表格、表单、弹窗主体       |
| 边框灰   | `#E5E7EB` | 表格线、输入框边框、分割线 |
| 正文黑   | `#1F2937` | 主要文字                   |
| 次级灰   | `#6B7280` | 辅助说明                   |
| 弱灰     | `#9CA3AF` | 占位符、弱提示             |
| 成功     | `#22C55E` | 成功、完成、正常状态       |
| 警告     | `#F59E0B` | 待确认、部分完成、风险提示 |
| 错误     | `#EF4444` | 失败、异常、危险操作       |
| 信息     | `#3B82F6` | 进行中、信息提示           |

使用原则：以白色、浅灰和深蓝为主；黄色仅作少量强调；危险操作使用红色或危险样式；同一状态语义在全系统复用相同颜色；表格状态优先使用标签而非大面积色块。

## 3. 字体与排版

```css
font-family: Inter, 'PingFang SC', 'Microsoft YaHei', 'Noto Sans SC', Arial, sans-serif;
```

| 场景     | 字号 | 字重 |
| -------- | ---: | ---: |
| 页面标题 | 20px |  600 |
| 区块标题 | 16px |  600 |
| 表格正文 | 14px |  400 |
| 表单标签 | 14px |  500 |
| 辅助说明 | 12px |  400 |
| 状态标签 | 12px |  500 |

页面内容边距为 24px；卡片内边距为 16px 或 20px；表单垂直间距为 16px；按钮间距为 8px；工具栏与表格间距为 12px。

## 4. 管理端布局

管理端使用固定后台结构：左侧菜单、顶部栏和内容区。顶部栏承载系统级信息、面包屑、当前用户和退出操作；不要堆放业务操作按钮。

左侧菜单建议宽度 220px，最多两级，使用轻量线性图标；当前菜单使用深蓝背景或蓝色左侧高亮。实际菜单项由路由和权限配置驱动，不在本规范中固定。

内容区优先使用“页面标题 + 可选说明 + 筛选/工具栏 + 表格或卡片 + 分页”的信息层级；业务需要时可使用详情区域或卡片分组，但不绑定具体页面类型。

## 5. 弹窗规范

新增、编辑、查看详情、分配、确认和上传等操作统一使用 **Modal**；不使用 Drawer 作为主要交互方式。

弹窗按内容复杂度自适应，最大宽度不超过 `75vw`、最大高度不超过 `85vh`。内容溢出时只在弹窗内部滚动。

| 类型       | 推荐宽度 | 最大宽度 |
| ---------- | -------: | -------: |
| 小型确认   |    420px |     90vw |
| 常规表单   |    640px |     75vw |
| 多字段表单 |    860px |     75vw |
| 复杂详情   |   1000px |     75vw |

```css
.dialog-sm {
  width: min(420px, 90vw);
}
.dialog-md {
  width: min(640px, 75vw);
}
.dialog-lg {
  width: min(860px, 75vw);
}
.dialog-xl {
  width: min(1000px, 75vw);
}
.dialog-body {
  max-height: 70vh;
  overflow-y: auto;
}
```

弹窗底部取消按钮在左、确认按钮在右；确认文案必须表达实际动作。删除、取消、关闭、作废等危险操作必须二次确认；关闭已修改的表单时应提示是否放弃修改。

## 6. 表格规范

表头使用 `#F9FAFB`，边框使用 `#E5E7EB`，建议行高 48px，悬停行背景使用 `#F3F4F6`。关键识别字段可加粗。

列顺序遵循：识别字段 → 名称/型号 → 分类/类型 → 数量 → 状态 → 负责人 → 时间 → 操作。行内操作应保持克制，超过四项时收入“更多”；危险操作进入“更多”或二次确认弹窗。

## 7. 表单规范

简单表单使用单列，中等复杂度表单使用两列，复杂表单通过通用分组标题组织。必填项必须清晰标识。

| 数据类型  | 优先控件       |
| --------- | -------------- |
| 文本      | Input          |
| 长文本    | Textarea       |
| 枚举/分类 | Select         |
| 数量      | InputNumber    |
| 日期      | DatePicker     |
| 日期时间  | DateTimePicker |
| 文件      | Upload         |
| 布尔值    | Switch         |
| 多选      | CheckboxGroup  |

接口成功后的新增、编辑、删除、确认、上传、下载等操作必须经 `EMessage` 提示成功；失败时必须显示可理解的失败原因。查询列表可保持静默。

## 8. 图标、圆角与空状态

使用轻量线性图标，不使用复杂插画。卡片圆角 8px，按钮和输入框圆角 6px，弹窗圆角 10px；卡片使用轻微或无阴影，弹窗可使用中等阴影。

空状态应包含简短提示、可选说明和一个与当前上下文相关的主要操作；不硬编码未来业务模块或功能名称。

## 9. 响应式与交付检查

一期优先适配桌面端，主要适配宽度不低于 1440px，最低支持 1280px；窄屏下表格允许横向滚动，不优先为手机端重构管理端。

页面交付前检查：

- 是否使用统一色彩、字体、间距和状态语义；
- 是否使用 Modal 而非 Drawer；
- 是否为危险操作提供二次确认；
- 是否为业务操作提供 `EMessage` 成功/失败反馈；
- 是否避免把具体页面规划、菜单规划和业务状态字典写入本规范；
- 是否检查中文文案为 UTF-8，且不存在乱码。
